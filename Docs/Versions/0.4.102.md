# Liberation OS 0.4.102

- Added a dedicated x64 kernel diagnostics subsystem in `Source/Src/Arch/X64/Kernel/C/Diagnostics.c` and `Source/Src/Arch/X64/Kernel/H/Diagnostics.h`.
- Enabled compiler-driven function-entry instrumentation for the kernel, interrupt, and memory C units so entered functions are recorded through the diagnostics subsystem without hand-adding trace calls everywhere.
- Initialised diagnostics at higher-half kernel entry and redirected fatal runtime exceptions through the new diagnostics crash reporter.
- Updated runtime fault diagnostics to print visible `[FAIL]`-tagged fault decode lines, the full general register dump, CR2/CR3/CR4, and the 32 bytes at RIP so the serial line filter no longer hides the most important crash details.
- Added recent-function-entry crash trace output so a fatal exception now ends with the latest recorded call-path addresses.
