# 0.4.69

- Fixed memory-manager request authorisation across mixed trees by accepting the implicit kernel bootstrap caller as well as the explicit `kernel` principal.
- Marked kernel bootstrap MM requests with the bootstrap principal type so post-attach frame allocation no longer falls back to the direct-claim scheduler stack pool.
- Updated both ChangedFiles and FullSource copies of the MM dispatch path so rebuilding from either tree produces the same hosted-bootstrap behaviour.
