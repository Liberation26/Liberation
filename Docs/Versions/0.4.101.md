# Liberation OS 0.4.101

- Fixed the x64 interrupt-entry assembly to realign the kernel stack before calling the C interrupt handler, so fatal exception diagnostics can safely print the full register set and RIP bytes instead of stopping after the first two lines.
- Fixed the scheduler user-transition bridge assembly to realign the stack before calling `LosKernelSchedulerPrepareUserTransitionIret()`, removing another fragile call site on the first CPL3 handoff path.
- Kept the rest of the CPL3 proof path unchanged so the next boot log should now tell us whether the remaining fault is still in the handoff itself or only in diagnostics.
