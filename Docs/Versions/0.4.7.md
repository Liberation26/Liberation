# Version 0.4.7

This upgrade moves the shell bootstrap path from direct service-name bind requests to a registry-style resolve step followed by transport binding.

Highlights:
- added shell registry-entry, resolve-request, and resolve-response ABI records
- shell now publishes a registry-visible service identity with a stable bootstrap service id
- `InitCommand` resolves the shell first, then binds using the resolved generation
- keeps the shared-mailbox bootstrap transport intact while making the later swap to kernel-managed service registration smaller and cleaner
