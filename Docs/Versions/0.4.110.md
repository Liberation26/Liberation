# Liberation OS 0.4.110

- Reworked the first-user-task bridge staging to record direct kernel function addresses instead of routing the bridge and kernel-entry stack slots through helper-returned runtime addresses.
- Added hard validation of the live user-transition contract just before the real `iretq`: the handoff stack slots, bridge address, kernel-entry address, user RIP, user RSP, and prepared frame must all match before the transition is allowed to proceed.
- Added deeper dispatch diagnostics around the live user-task bridge handoff so the exact failing slot or frame field is printed immediately if the transition contract is broken again.
- Kept the version string unsuffixed and bumped the project version to `0.4.110`.
