# Liberation OS 0.4.97

- Fixed the scheduler first-user-task bridge stack layout so the bridge return address is written one slot below the kernel-entry return address instead of overwriting the same stack slot.
- The bridge-ready stage now decrements the task execution stack pointer by 8 bytes before writing the bridge address, leaving the expected two-step return chain as `[SP+0] = Bridge` and `[SP+8] = KernelEntry`.
- This directly addresses the invalid-opcode fault after the 200-tick heartbeat where the first CPL3 handoff could return through an uninitialised second stack slot instead of landing on the kernel entry path.
