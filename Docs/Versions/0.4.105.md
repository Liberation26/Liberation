# 0.4.105

- Fixed the first user-task ring-3 handoff so the scheduler now writes the bridge and kernel-entry return chain into the dedicated chain stack slot instead of reusing the near-top bootstrap return slot.
- Switched the user-transition bridge and kernel-entry address capture to RIP-relative runtime helpers so the scaffold uses the live runtime address of those entry points.
- Tightened contract readiness so the scaffold refuses to go live unless the prepared iret frame and the return-chain slots match the expected user RIP, selectors, RFLAGS, and stack pointer.
