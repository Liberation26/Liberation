# LOS 0.4.21

- the shell runtime now tries a disk-backed image path before bootstrap dispatch by reading the target ELF from `\LIBERATION\...`, validating it as an `ELF64 ET_EXEC` image, and building a `LOS_USER_IMAGE_EXECUTION_CONTEXT`
- added weak shell hooks for disk image read and loaded-image execution so the real filesystem and memory-manager launch path can plug in without changing the shell ABI again
- if the loaded-image execution hook is not wired yet, the shell now falls back to the existing bootstrap image dispatcher so login and string calls still work during bring-up
- the standalone login command and string library `_start` entry points now also accept a `LOS_USER_IMAGE_CALL *` passed in `rdi`, preparing them for true loader-driven structured invocation
