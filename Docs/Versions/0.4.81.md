# LOS 0.4.81

- Reduced the scheduler lifecycle-manager period from 250 ticks to 25 ticks so the first-user-task bring-up ladder advances quickly instead of pausing for multiple seconds between each stage.
- Moved the staged first-user-task chain stack and synthetic initial `iretq` frame much deeper into the kernel task stack, leaving substantially more headroom for the first user-mode IRQ/trap entry and its kernel-side call chain.
- This targets the remaining reboot/reset loop that still appeared immediately after `marked live for real iretq dispatch` even after the earlier stack-offset hardening work.
