# LOS 0.4.11

## Summary

This update changes login authorisation so the external login path is allowed or denied by the capability service instead of by a hardcoded user check in the shell bootstrap adapter.

## Changes

- added `session.login` capability grants to the administrator and administrator.programmer profiles
- added a `service.login` profile assignment for the external login command
- updated the bootstrap login fallback to query the capability service for `session.login`
- updated the external `LOGIN.ELF` source so its authentication contract is capability-driven
- changed the bootstrap shell demo login from `root` to `dave`

## Effective behaviour

A login now succeeds only when both of these are true:

1. the password matches the current bootstrap secret
2. the capability service grants the user `session.login`

If the capability check fails, the shell stays at `login>` and returns an access-denied result.
