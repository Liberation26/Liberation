# Liberation OS 0.4.95

- Replaced the kernel scheduler's embedded first user-task proof image with a tiny freestanding static ELF that enters ring 3, raises `int 0x80`, and then idles if control returns unexpectedly.
- Staged that image into ordinary claimed RAM before asking the memory manager to attach it, instead of passing the memory manager a physical pointer into the kernel image's own embedded byte array.
- Added first user-task staging diagnostics so attach failures now report the staging status, physical base, and staged byte count.
- This update is aimed directly at the repeated "could not attach the first user-task image" failure so the CPL3 proof path can advance cleanly into the next bootstrap stage.
