# 0.4.67

- Fixed the post-bootstrap frame-ownership hole between the kernel scheduler and the memory manager.
- The scheduler stack-pool reservation now claims frames through the memory-manager transport once MM attach is complete, so those physical pages are recorded in the MM page-frame database before later staged-image work runs.
- This prevents the staged-image attach path from reusing the scheduler stack-pool range and triggering the low-address `#UD Invalid Opcode` fault immediately after `attach-return-success`.
