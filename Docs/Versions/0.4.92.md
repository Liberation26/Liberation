# LOS 0.4.92

This update moves the proof-mode login flow under the shell itself so the shell clears the screen, runs the external login command, and collects the user's credentials before entering the command loop.

## Summary
- bumped the archive version to 0.4.92
- moved bootstrap console ownership for login back into the shell instead of init driving synthetic shell commands
- made the shell clear the screen before prompting for credentials
- added proof-mode input collection for username and password on the bootstrap COM1 console
- kept login execution on the existing external `LOGIN.ELF` path so capability-based authentication still decides success

## Why this change
The shell should own the login experience. The previous proof path still had init sending canned commands into the shell, which meant login was not being started from the shell in the way the design expects. This update makes the shell start the session itself, collect the user's input, and then continue into the interactive prompt once authentication succeeds.
