# LOS 0.4.25

- changed the login command and string library standalone entry points to return directly through `_start(const LOS_USER_IMAGE_CALL *Call)` so the disk-backed loader can call them with the real image ABI
- removed the fallback standalone `int 0x80` entry path from the login and string user images so they build as freestanding callable images without interpreter expectations
- aligned the login and string linker scripts to fixed executable bases intended for direct `ET_EXEC` loading
- extended the user image execution context with stack address and size fields and staged a dedicated runtime stack buffer in the shell runtime before invoking the loaded-image executor
- updated the build script and package version metadata for the 0.4.25 loader handoff
