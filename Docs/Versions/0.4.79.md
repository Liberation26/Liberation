# 0.4.79

- Prefer memory-manager-backed scheduler thread stacks once the memory-manager transport is online, using the direct-claim pool only as a fallback.
- Stop the scheduler lifecycle from spawning transient ephemeral worker processes while the first live user task is pending its initial real dispatch.
- This removes the repeated direct-claim stack-pool status spam during the final handoff window and keeps the launch path focused on the first user-mode task.
