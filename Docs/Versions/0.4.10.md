# LOS 0.4.10

- user command images no longer carry the `X64` suffix on disk
- `login` now resolves to `\LIBERATION\COMMANDS\LOGIN.ELF`
- added public `LoginCommandAbi.h` for explicit external login results
- shell no longer treats launcher success as authentication success
- shell sets `Authenticated = 1` only when the external login result is `AUTHENTICATED`
- failed login now returns access denied and leaves the prompt at `login>`
- added a bootstrap login adapter that validates `root / liberation` until the real exec-return path is wired
