# Liberation OS 0.4.99

- Kept the bridge-ready stack setup explicit by recording the bridge dispatch return target at a new lower stack slot and tracing both the bridge slot and the next kernel-entry slot when the bridge becomes ready.
- Hardened the first-user-task lifecycle so a COMPLETE scaffold can no longer be re-promoted back to LIVE; the compatibility shim and live-promotion gate now both require UserTransitionCompleteCount to remain zero.
- Added live-stage stack-slot tracing so the scheduler now reports the active live stack pointer plus the first two return slots immediately before the real iretq dispatch path.
