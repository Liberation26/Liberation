# Liberation OS 0.4.94

- Reworked first user-task interrupt handling so vector 128 completes the CPL3 proof path, while ordinary runtime interrupts can continue through the normal interrupt path instead of being treated as immediate completion.
- Added stronger first user-task fault diagnostics, including the fault vector, error code, RIP, RSP, expected entry, and user selectors.
- Replaced the embedded scheduler proof image with a safer ring-3 stub that returns through `int 0x80` and then idles with `pause` if the return unexpectedly resumes in user mode.
- This update focuses on getting a cleaner CPL3 proof signal and better fault evidence before moving on to full shell bootstrap wiring.
