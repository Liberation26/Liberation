# Liberation OS 0.4.13

- removed the stray timer-path text file from the package workspace
- added `Source/Include/Public/UserImageAbi.h` with a generic callable user-image contract
- shell runtime now builds `LOS_USER_IMAGE_CALL` records for both external commands and string-library calls
- `LosUserInvokeStringLibrary(...)` and `LosUserLaunchExternalCommand(...)` now share the same `LosUserLoadAndCallImage(...)` runtime hook
- the low-level loader hook still defaults to unsupported until the next loader stage is wired into the memory manager or scheduler path
