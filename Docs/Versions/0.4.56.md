# Liberation OS 0.4.56

- Expanded the memory-manager bootstrap offered operation bitmap so the hosted memory-manager service now negotiates and accepts `AttachStagedImage`, `AllocateAddressSpaceStack`, `QueryMemoryRegions`, `ReserveFrames`, and `ClaimFrames`.
- This fixes the runtime halt where the scheduler's first user-task image attach failed with status `3` (`NOT_SUPPORTED`) immediately after the memory-manager bootstrap completed.
- Kept packaged build output out of the release archive.


- Fixed memory-manager leaf page flag validation to accept PRESENT/USER bits used by staged-image and stack attach paths.
