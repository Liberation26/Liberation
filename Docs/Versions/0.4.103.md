# Liberation OS 0.4.103

- Fixed the immediate post-`ExitBootServices` page fault introduced by compiler function instrumentation by moving the diagnostics hooks and trace storage into bootstrap-mapped sections.
- Kept compiler-driven function-entry tracing enabled for kernel, interrupt, and memory C units, but made `__cyg_profile_func_enter` and the diagnostics trace buffer callable before the higher-half text mapping is live.
- Preserved the runtime crash reporter so fatal faults still print the full register dump, CR2/CR3/CR4, RIP bytes, and recent recorded function-entry history after higher-half entry.
- This change specifically addresses the observed instruction-fetch page fault at `FFFFFFFF800008F0`, which was the instrumentation hook being called before the higher-half diagnostics text had been mapped.
