# LOS 0.4.85

- Increased the default kernel-screen font scale and slightly tightened line spacing so the framebuffer console is readable again on the QEMU display.
- Stopped init from immediately returning to the kernel after the first successful ring-3 bootstrap sequence; on success it now remains resident in a quiet parked state instead.
- Clarified the bootstrap flow inside init: the memory manager is already online before init starts, while CAPSMGR and SHELL are still being brought up through the current proof-mode bootstrap path rather than through the final separate scheduled-service launcher.
