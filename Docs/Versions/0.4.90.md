# LOS 0.4.90

This update focuses on the timer path stalling around tick 140 after the first visible heartbeat.

## Summary
- bumped the archive version to 0.4.90
- kept the early boot scheduler in a conservative timer-proof mode by deferring the lifecycle manager and busy-worker bootstrap tasks
- left the heartbeat task enabled so the scheduler can still report once the serial filter allows it
- tagged exception and unhandled runtime interrupt serial lines with `[FAIL]` so faults are visible even through the current filtered console path

## Why this change
The timer was continuing long enough to prove IRQ0 was alive, then stopping once the scheduler started bringing in higher-level bootstrap work. That points to the early auxiliary scheduler tasks rather than the PIT itself. This change keeps the kernel parked on the stable idle-plus-heartbeat path until the timer proof remains solid.
