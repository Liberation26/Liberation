# Version 0.4.6

This upgrade moves the shell bootstrap path from direct transport binding to service-name discovery plus bind requests.

Highlights:
- added shell discovery record and bind-request ABI records
- shell publishes a discovery record alongside the transport export
- `InitCommand` now binds the shell by service name (`shell`) and expected ABI version
- keeps the shared-mailbox bootstrap transport intact while cleaning the lookup boundary for later kernel-managed service registration
