# Liberation OS 0.4.104

- Removed the bootstrap trap reporter from the default x64 kernel handoff path so early post-`ExitBootServices` bootstrap stays minimal.
- Stopped compiling the kernel, interrupt, and virtual-memory C units with compiler-wide function instrumentation, which removes the `__cyg_profile_func_enter` hook from the higher-half entry path.
- Moved the kernel diagnostics state back out of bootstrap-only sections so diagnostics belong to the real kernel runtime rather than the temporary bootstrap mapping.
- Kept the x64 runtime crash reporter in place so later fatal exceptions still print the fault decode, register dump, CR2/CR3/CR4, RIP bytes, and any recorded diagnostics state once the higher-half kernel is running.
