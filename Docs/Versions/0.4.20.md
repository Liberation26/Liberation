# LOS 0.4.20

- the shell runtime now dispatches `LosUserLoadAndCallImage(...)` through a bootstrap image registry for `\\LIBERATION\\COMMANDS\\LOGIN.ELF` and `\\LIBERATION\\LIBRARIES\\STRING.ELF`
- added structured bootstrap invoke entry points in the login command and string library so both can consume `LOS_USER_IMAGE_CALL` request/response blocks directly
- `BuildBoot.sh` now compiles the login command and string library a second time as shell-embedded bootstrap-callable objects, while still producing their standalone ELF images
- this keeps one generic image-call ABI in place now, while the true disk-backed loader path is still pending
