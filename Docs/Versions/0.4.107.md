- 0.4.107: added full kernel-side EFI memory-map and normalized memory-region dumping, added memory-manager launch-map dumping, and made the active interrupt-controller path explicit as legacy PIC plus PIT while LAPIC/IOAPIC remain disabled.
- The kernel now prints every EFI memory descriptor after higher-half entry, then prints the normalized region map it derived for paging and handoff.
- The memory-manager bootstrap diagnostics now print the exact normalized region table passed to the memory manager so its view of memory is visible even before the real service console path exists.
- Boot logging now states explicitly that the active timer and interrupt path is legacy 8259 PIC plus PIT, not LAPIC or IOAPIC.
